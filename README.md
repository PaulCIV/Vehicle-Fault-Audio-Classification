An end-to-end machine learning pipeline for classifying mechanical vehicle faults from real-world audio recordings using classical machine learning, deep learning, Transformer architectures, and generative data augmentation.

# Overview

This project explores automatic vehicle fault diagnosis using acoustic recordings collected from real vehicles and diagnostic videos.

Unlike many academic datasets, the data used in this project contains realistic background noise, varying recording quality, and limited labeled examples. The goal was to investigate how different machine learning approaches perform under these real-world constraints while evaluating whether synthetic data generated by a Conditional Variational Autoencoder (cVAE) could improve classification accuracy.

The project includes the complete machine learning workflow:

- Data collection
- Audio preprocessing
- Feature extraction
- Classical machine learning baselines
- Deep learning models
- Transformer models
- Synthetic data generation
- Cross-validation
- Experimental evaluation

# Motivation

Experienced mechanics can often identify mechanical problems simply by listening to a vehicle.

This project investigates whether modern machine learning models can learn these same acoustic patterns from noisy real-world recordings and classify common vehicle faults before catastrophic failures occur.

Potential long-term applications include:

- Preventive maintenance
- Automated diagnostics
- Driver assistance
- Fleet monitoring
- Predictive maintenance systems

# Data Processing Pipeline

The preprocessing pipeline transforms raw audio into machine-learning-ready spectrograms.

Pipeline stages include:

- Audio resampling
- Mono conversion
- Sliding-window segmentation
- Energy-based filtering
- Loudest-region extraction
- Mel spectrogram generation
- Normalization
- Group assignment for evaluation

The preprocessing pipeline was designed specifically to reduce data leakage and maximize useful signal from noisy recordings.

# Machine Learning Models

Multiple model families were evaluated and compared.

## Classical Machine Learning

- k-Nearest Neighbors (kNN)
- Linear Discriminant Analysis (LDA)
- Support Vector Machines (SVM)

These models provide baseline performance for comparison against deep learning approaches.

## Deep Learning

### Convolutional Neural Network (CNN)

The CNN processes log-scaled Mel spectrograms as images, learning local time-frequency representations of mechanical sounds.

### Transformer

The Transformer processes spectrograms as sequences of frequency vectors, allowing long-range temporal relationships to be learned directly through self-attention.

The Transformer consistently outperformed the CNN on the collected dataset.

## Synthetic Data Generation

To address limited data availability and class imbalance, a Conditional Variational Autoencoder (cVAE) was implemented.

The cVAE learns a latent representation conditioned on vehicle fault classes and generates synthetic Mel spectrograms for augmentation experiments.

Rather than assuming synthetic augmentation would improve performance, the project experimentally evaluates its effect.

# Experimental Design

A major focus of this project was producing realistic evaluation results.

To prevent data leakage, all experiments use **Group K-Fold Cross Validation**, ensuring that windows originating from the same source recording never appear in both training and testing sets.

This provides a significantly more realistic estimate of model generalization than randomly splitting spectrogram windows.

# Key Findings

The experiments produced several important observations.

### Transformer vs CNN

The Transformer demonstrated stronger generalization than the CNN on real-world vehicle recordings.

### Synthetic Data

One of the primary research questions investigated whether synthetic spectrogram generation improves classification.

Results showed:

- Large quantities of synthetic data degraded performance.
- Smaller amounts produced mixed results depending on the model.
- Distribution mismatch between generated and real spectrograms limited the effectiveness of augmentation.

This finding highlights an important practical limitation of generative augmentation when working with small, noisy datasets.

# Challenges

Key engineering challenges included:

- Building a labeled dataset from scratch
- Handling noisy real-world recordings
- Designing leakage-resistant evaluation
- Managing severe class imbalance
- Implementing multiple model architectures
- Comparing fundamentally different learning approaches
- Evaluating generative augmentation objectively

# Lessons Learned

This project reinforced several important machine learning principles:

- Dataset quality often matters more than model complexity.
- Preventing data leakage is critical for meaningful evaluation.
- Larger models do not guarantee better performance.
- Synthetic data must closely match the true data distribution to improve generalization.
- Real-world datasets introduce challenges rarely present in benchmark datasets.
